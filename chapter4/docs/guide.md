# 開発指針

## コーディング規約
- Go標準のフォーマッタ (`gofmt`) に従う
- エラーは必ずハンドリングし、`_` で握りつぶさない
- パッケージ外に公開する型・関数にはGoDocコメントを付与する

## 命名規則
- パッケージ名: 小文字の単一単語 (例: `client`, `model`)
- ファイル名: スネークケース (例: `github_client.go`)
- 変数・関数名: キャメルケース (Go標準に準拠)
- 定数: キャメルケース (Go標準に準拠)

## gitコミット粒度
- 機能単位でコミットする (例: 「リポジトリ情報取得機能を追加」)
- テストコードは機能実装と同じコミットに含める
- コミットメッセージは日本語で簡潔に記述する
